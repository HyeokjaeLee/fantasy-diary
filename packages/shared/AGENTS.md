# Shared Package 개요

이 패키지는 `apps` 하위의 여러 어플리케이션(`web`, `agent-server`)에서 공통으로 재사용되는 로직과 유틸리티 함수들을 관리합니다.

## 역할 및 목적

- **코드 재사용**: 중복되는 비즈니스 로직이나 유틸리티를 한곳에서 관리하여 유지보수 효율을 높입니다.
- **일관성 유지**: 전역적으로 사용되는 타입, 유효성 검사 규칙(Zod 등)을 통일하여 애플리케이션 간 데이터 정합성을 보장합니다.

## 포함 내역

- **유틸리티 함수**: 날짜 포맷팅, 문자열 조작, 일반적인 헬퍼 함수 등.
- **도메인 로직**: 소설 생성 관련 공통 알고리즘, 데이터 가공 로직.
- **API 클라이언트 및 타입**: Supabase 클라이언트 설정, 공통 데이터베이스 타입 및 DTO.
- **유효성 검사**: Zod 스키마를 통한 공통 데이터 검증 로직.

## 개발 가이드라인

- **순수 함수 지향**: 가능한 부작용(Side Effect)이 없는 순수 함수로 작성하여 테스트와 재사용이 용이하게 하세요.
- **독립성 유지**: `apps` 하위 패키지에 의존하지 않도록 설계해야 합니다. 오직 `packages/dev`와 외부 라이브러리에만 의존할 수 있습니다.
- **명시적 Export**: 외부에서 사용할 기능은 `src/index.ts`를 통해 명시적으로 노출하세요.
- **타입 정의**: TypeScript의 이점을 최대한 활용하여 명확한 타입 정의를 제공하세요.
