# Web App 개요

이 워크스페이스는 사용자가 직접 일기를 작성하고 AI가 생성한 소설을 감상하는 **프론트엔드 어플리케이션**입니다.

## 역할 및 목적

- **사용자 경험(UX) 제공**: 판타지 컨셉에 몰입할 수 있는 인터페이스를 제공하며, 일기 작성부터 소설 발행 확인까지의 전 과정을 담당합니다.
- **데이터 상호작용**: 사용자의 입력을 서버로 전달하고, 생성된 소설 데이터를 시각화하여 보여줍니다.

## 기술 스택 및 배포

- **Framework**: **Next.js 16** (App Router 지향)
- **Runtime**: Bun
- **Deployment**: **Vercel**
  - Git Push 기반의 자동 배포 워크플로우를 사용합니다.
  - Vercel Analytics 및 Speed Insights를 통해 성능을 모니터링합니다.
- **Styling**: Tailwind CSS
- **Communication**: tRPC, TanStack Query

## 개발 가이드라인

- **Vercel 최적화**: 서버 사이드 렌더링(SSR)과 정적 재생성(ISR)을 적절히 활용하여 로딩 속도를 최적화하세요.
- **모바일 우선(Mobile-first)**: 일기 서비스 특성상 모바일 접근성이 중요하므로, 반응형 레이아웃을 반드시 준수하세요.
- **접근성(Accessibility)**: 모든 사용자가 소설을 편하게 읽을 수 있도록 웹 접근성 표준을 따르세요.
- **환경 변수**: Vercel 대시보드에서 관리되는 환경 변수를 사용하며, 로컬 개발 시에는 `.env.local`을 활용하세요.
