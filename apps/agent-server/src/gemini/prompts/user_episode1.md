대상 소설 ID: {{novelId}}
이번에 작성할 회차 번호: {{episodeNo}} (첫 회차)

규칙(매우 중요):
- (도입) 첫 문단은 반드시 **인물의 행동/선택/대사**로 시작하라. (배경/날씨/분위기(어둠/추위/바람/정적 등)로 시작 금지)
- (도입) 첫 2문단은 **지금 당장 벌어지는 사건/위협/선택**만으로 구성하라.
  - 프롤로그/세계관 소개/장황한 상황 정리로 리셋하며 시작 금지.
  - 설명문/설정 나열 금지. 장면 속 행동과 대사로 보여라.
- (신규 요소) 첫 회차는 **새 인물/새 고유명사/새 장소**를 소개할 수 있다.
  - 단, 한꺼번에 많이 소개하지 말고(인물 1~2명, 장소 1곳 수준), 정보는 행동/대사/관찰을 통해 자연스럽게 드러내라.
- (시간 표현) 본문에 숫자 시각 표기 금지: "오전/오후 N시", "N시 N분", "HH:MM" 같은 직접적인 시각은 쓰지 마라.
  - 시간 경과는 "잠시 후/얼마 후/어느새/주변이 더 어두워진 뒤" 같은 상대 표현으로 처리.
- (메타 금지) 회차 번호/회차 라벨(예: 1회차, 1화, 지난 회차) 언급은 금지다.
- (분량) 상위 컨텍스트(수정 지시)에 주어진 분량 범위는 **하드 제한**이다. 반드시 범위 안으로 맞춰라.

필수: Supabase에서 `novels.story_bible`과 `novels.append_prompt`를 확인하고, story_bible을 최우선으로 준수하되 append_prompt 지시를 추가로 따른다.

JSON만 출력.

추가 출력(선택): 가능하면 JSON에 entities 필드를 포함해라.
- entities는 (characters/locations/plot_seeds) 3개 배열을 가진 객체다.
- 근거 없는 생성 금지(본문/제공된 DB 스냅샷 밖의 정보는 쓰지 마라).
- 이름이 없으면 name=null + descriptor로 기록(임의로 이름 만들지 마라).
